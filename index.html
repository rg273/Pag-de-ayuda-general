Configuraciones

git config --global user.email

 cambia de manera global el email del usuario, seguidamente ponemos entre comillas nuestro nombre.

git config --list

 muestra configuraciones de git también podemos usar --show-origin inhales nos muestra las configuraciones guardadas y su ubicación.

Comandos Basicos (Git del dia a dia de un programador)

1° git init

 lo usamos para determinar la carpeta en la que vamos a trabajar.

git status

 lo usamos para saber si tenemos un archivo añadido o borrado en nuestro proyecto, para saber en la rama en la que estamos y si tenemos commits.

4° git add .

 es para añadir todos los archivos que tengan cambios a nuestra rama o remplazar el punto por el nombre de nuestro archivo, por ejemplo:
    git add ArchivoEjemplo.js

git rm

 lo usamos para borrar un archivo que hayamos añadido, eliminarlo por completo de nuestra rama usamos git rm --cached.

5° git commit -m "Aca_Debe_Ir_Nuestro_Mensaje"

 se usa para añadir un mensaje a nuestra rama. Esto nos es util para avisar a los otros programadores los cambios que acabamos de hacer a los archivos rama

git log

 se usa para ver la historia de nuestros archivos, los commits, el usuario que hizo los cambios al archivo, cuando se realizaron los cambios etc.


Fusionar ramas en local

git merge (nombre rama)

Fusiona una rama (nombre_rama) con la actual(rama que nos encotramos posicionado)

git merge (nombre rama) --allow-unrelated-histories:

lo mismo que la anterior solamente que ahora lo obliga

Comandos que van para repo local y remoto

2° git remote add origin (ssh o https)

se usa para conectar el repositorio remoto con nuestro repositorio local(carpeta a la que le aplicamos el git init).

3°(opcional) git remote -v

se usa para ver las conexiones existentes que hay con el repo remoto.

git remote remove origin

Se usa para eliminar una conexion con el repositorio remoto.

7° git push origin main

se usa para enviar los cambios commiteados al github.

6°(opcional) git pull origin main

descarga los cambios del repo remoto y los fuciona a (main) del local.

git pull origin main --allow-unrelated-histories

Obliga a fucionar la rama de origin (main) del repo REMOTO con la actual (con la rama en la que estemos posicionados del local)
